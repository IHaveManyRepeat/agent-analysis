# OpenHarness 架构分析

## 1. 职责概述

OpenHarness 是一个轻量级的 AI 代理基础设施，提供工具使用、技能管理、内存管理和多代理协调等核心功能。它的主要职责包括：

- **代理循环管理**：处理流式工具调用循环、API 重试、并行工具执行和令牌计数
- **工具集管理**：提供 43+ 种工具，包括文件 I/O、shell、搜索、Web 和 MCP 等
- **技能系统**：支持按需加载技能（.md 文件），兼容 anthropics/skills
- **插件生态**：兼容 claude-code 插件，支持命令、钩子、代理和 MCP 服务器
- **权限管理**：实现多级权限模式、路径级规则和命令拒绝列表
- **多代理协调**：支持子代理生成和委托、团队注册和任务管理
- **个人代理应用**：提供 ohmo 个人 AI 代理，支持 Feishu/Slack/Telegram/Discord 等平台

## 3. 对外接口

### CLI 命令

- `oh`：主命令，启动 OpenHarness
- `oh setup`：交互式设置向导
- `oh provider`：管理 LLM 提供商配置
- `oh auth`：认证管理
- `oh mcp`：MCP 服务器管理
- `oh plugin`：插件管理
- `ohmo`：个人代理应用命令

### 核心 API

- `QueryEngine`：代理循环核心，处理模型查询和工具调用
- `ToolRegistry`：工具注册和管理
- `SkillLoader`：技能加载和管理
- `PermissionChecker`：权限检查
- `SwarmCoordinator`：多代理协调
- `MemoryManager`：内存管理
- `ApiClient`：LLM API 客户端

## 5. 代码质量

### 优点

- **模块化设计**：清晰的模块划分，职责明确
- **类型提示**：全面的类型标注，提高代码可读性和可维护性
- **测试覆盖**：114 个单元/集成测试，6 个 E2E 测试套件
- **文档完善**：详细的 README 和文档
- **安全设计**：多级权限模式和路径验证

### 改进空间

- **文件大小**：部分核心文件如 `query_engine.py` 接近 300 行，可考虑进一步拆分
- **依赖管理**：部分依赖版本固定，可考虑使用更灵活的版本约束
- **代码重复**：不同工具实现之间存在一定的重复代码，可考虑提取公共逻辑

## 6. 改进建议

### 1. 模块化拆分

- **问题**：核心引擎文件较大，职责较为集中
- **建议**：将 `query_engine.py` 拆分为更小的模块，如 `stream_processor.py`、`tool_executor.py` 等
- **预期效果**：提高代码可维护性和测试覆盖
- **实施复杂度**：中

### 2. 依赖管理优化

- **问题**：部分依赖版本固定，可能导致兼容性问题
- **建议**：使用更灵活的版本约束，如 `^1.0.0` 而非 `==1.0.0`
- **预期效果**：提高系统兼容性和可升级性
- **实施复杂度**：低

### 3. 工具抽象优化

- **问题**：工具实现之间存在重复代码
- **建议**：提取工具执行的公共逻辑，创建更高级的抽象
- **预期效果**：减少代码重复，提高可维护性
- **实施复杂度**：中

### 4. 性能优化

- **问题**：大型技能和插件加载可能影响启动速度
- **建议**：实现技能和插件的懒加载机制
- **预期效果**：提高启动速度和运行效率
- **实施复杂度**：中

### 5. 扩展性增强

- **问题**：部分核心功能扩展性有限
- **建议**：增加更多的钩子点和扩展接口
- **预期效果**：提高系统的可扩展性和可定制性
- **实施复杂度**：中

## 7. 架构特点

### 核心优势

1. **轻量级设计**：核心代理循环简洁明了，易于理解和扩展
2. **多提供商支持**：支持多种 LLM 提供商，包括 Anthropic、OpenAI、Codex 等
3. **丰富的工具集**：43+ 种工具，覆盖文件操作、网络搜索、代码分析等多种场景
4. **强大的技能系统**：支持按需加载技能，兼容 anthropics/skills
5. **多代理协调**：支持子代理生成和团队协作，提高复杂任务的处理能力
6. **多平台支持**：通过 ohmo 提供 Feishu、Slack、Telegram、Discord 等平台的支持

### 应用场景

- **代码助手**：阅读代码、修改文件、运行检查
- **自动化工具**：通过 json 和 stream-json 输出在自动化流程中使用
- **插件和技能测试**：实验 Claude 风格的扩展
- **多代理原型**：任务委托和后台执行
- **提供商比较**：在不同 Anthropic 兼容后端之间进行比较
