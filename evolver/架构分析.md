# EvolveR 架构分析

## 1. 职责概述

EvolveR 是一个通过经验驱动生命周期实现自我进化的 LLM 代理框架。它的主要职责包括：

- **经验管理**：从过去的轨迹中提取抽象原则，并将其存储在经验向量数据库中
- **自我进化**：通过闭环学习过程不断改进代理的能力
- **检索增强**：利用检索系统获取相关信息，辅助代理决策
- **奖励机制**：基于多种奖励信号评估代理行为和经验价值
- **模型训练**：使用 GRPO 等算法训练自我进化的 LLM 模型

## 3. 对外接口

### 主要脚本

- `scripts/train_grpo-3b.sh`：GRPO 训练主脚本
- `scripts/retrieval_launch.sh`：启动检索服务器
- `scripts/vllm_server.sh`：启动 VLLM 推理服务器
- `scripts/download.py`：下载数据和模型

### 核心 API

- `ExperienceManager`：经验管理核心，处理经验提取、存储和检索
- `PrincipleClient`：原则提取客户端，从经验中提取抽象原则
- `RetrievalSystem`：信息检索系统，获取外部知识
- `RewardCalculator`：奖励计算模块，评估代理行为
- `PPOTrainer`：PPO 训练器，训练自我进化的模型

## 7. 架构特点

### 核心优势

1. **经验驱动设计**：通过经验提取和重用实现自我进化
2. **多模态检索**：集成多种检索方法，增强代理知识获取能力
3. **综合奖励机制**：基于多种奖励信号评估代理行为
4. **灵活的训练框架**：支持多种训练算法和模型架构
5. **模块化架构**：各组件松耦合，易于扩展和定制

### 技术创新

1. **经验向量数据库**：使用 Milvus 存储经验向量，实现高效检索
2. **抽象原则提取**：从具体经验中提取抽象原则，实现知识泛化
3. **闭环学习**：通过经验-提取-应用的循环，实现持续进化
4. **GRPO 训练**：结合 GPT-4 和 PPO 的优势，提高训练效果

## 8. 应用场景

- **知识密集型任务**：如问答、信息检索、知识图谱构建
- **复杂推理任务**：如数学推理、逻辑推理、问题求解
- **自我改进系统**：需要持续学习和进化的 AI 系统
- **研究平台**：用于研究 LLM 自我进化的机制和方法
